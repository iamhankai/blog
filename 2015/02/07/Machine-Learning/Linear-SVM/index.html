手写数字识别问题

我们做过kNN分类算法识别手写数字的程序，但有个缺点，kNN需要保留所有的训练数据，而且每次识别都要遍历所有训练数据，这对于应用来说过于庞大，我们需要更精简的一种算法，它就是SVM。SVM只需要保留支持向量，而且分类效果也很好。
这次解决的问题和kNN识别手写数字一样，只是SVM是一个二类分类算法，我们只识别数字0和9，其他数字全部删除。

线性可分SVM分类器

对于线性可分数据集，我们构造一个超平面，也就是分类的决策边界，分布在超平面一侧的所有数据都属于某个类别，而分布在另一侧的所有数据则属于另一个类别。超平面的数学表达式可以写为
$${w^T}x + b$$
那么分布在超平面两侧的点代入上式，得出的值的正负相反，可以把数据集分为+1和-1两类。那么我们的分类决策函数就可以表示为
$$f(x) = {\rm{sign}}({w^T}x + b)$$
所以，只要找到超平面，或者说参数$w$、$b$，就得到了线性可分SVM。

实验结果

运行svm_smo中的testDigits，得到结果为186个测试点，2个识别错误，错误率为0.010753。效果和最好的kNN识别率相当。

总结

前面也提到过，kNN需要保留所有的训练数据，而且每次识别都要遍历所有训练数据，这对于应用来说过于庞大。而SVM只需要保留支持向量，分类效果也不亚于效果最好的kNN。因此，有些人认为，SVM是最好的现成的分类器，这里说的“现成”指的是分类器不加修改即可直接使用。
